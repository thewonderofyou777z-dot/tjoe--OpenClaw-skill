# 文档知识提取参考模板

## 提取流程

对文档内容按以下步骤提取知识：

### 1. 快速扫描（5分钟内）
- 文档类型、来源、日期
- 主题概述
- 核心人物/实体
- 关键结论（1-3句）

### 2. 深度提取（完整文档）

对每段/每节内容，提取：

```
## [实体/概念名称]

- **类型**: 人物 / 事件 / 概念 / 项目 / 产品 / 组织
- **关键信息**: 2-3句话描述
- **来源**: 文档名称 + 页码/章节
- **与其他实体的关系**: (关联到其他提取的实体)
- **重要程度**: ⭐⭐⭐ / ⭐⭐ / ⭐（高中低）
- **待验证/疑问**: 任何不确定的地方
```

### 3. 知识分类存储

提取后，按以下分类存储到对应文件：

| 知识类型 | 存储位置 |
|---------|---------|
| 用户偏好/习惯 | MEMORY.md 或 USER.md |
| 项目技术细节 | 项目目录下新建 `KNOWLEDGE.md` |
| 重要决策/结论 | MEMORY.md |
| 人物/公司/产品 | ontology/ 目录下按实体类型分 |
| 每日事件 | memory/YYYY-MM-DD.md |
| 通用知识 | memory/knowledge-base/ |

### 4. 输出格式

提取完成后，生成结构化摘要：

```markdown
# 文档知识提取报告

**文档**: [文件名]
**来源**: [URL/路径]
**日期**: [文档日期]
**提取时间**: YYYY-MM-DD HH:MM

## 摘要（3-5句话）

## 核心知识点

### 人物/实体
- ...

### 关键概念
- ...

### 重要事实
- ...

## 与现有知识的关联
- (对比 MEMORY.md，说明此文档更新/补充/修正了什么)

## 下一步行动
- (是否需要进一步研究/验证/应用)
```
